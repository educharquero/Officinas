📁 FileServer no Debian com binários

Tutorial de Configuração do Samba4 como FileServer no Debian 13

Este guia mostra como instalar e configurar o Samba4 em um servidor Debian 13, criando compartilhamentos de rede com permissões de usuários.

0 - Atualizar o Sistema

sudo apt update
sudo apt full-upgrade

1 - Instalar o Samba

sudo apt update
sudo apt install samba -y

2 - Criar usuários do sistema

Adicione os usuários que terão acesso:

sudo adduser oliveira
sudo adduser giovanna

3 - Definir senhas Samba para os usuários

Adicione os usuários ao banco de senhas do Samba:

sudo smbpasswd -a oliveira
sudo smbpasswd -a giovanna

4 - Criar diretórios para os compartilhamentos

Crie as pastas que serão compartilhadas:

sudo mkdir -p /arquivos/estoque
sudo mkdir -p /arquivos/gestao
sudo mkdir -p /arquivos/publico

5 - Definir permissões das pastas

Ajuste as permissões:

sudo chmod 777 -R /arquivos/estoque
sudo chmod 777 -R /arquivos/gestao
sudo chmod 777 -R /arquivos/publico

6 - Fazer backup do arquivo de configuração

Antes de editar, faça backup do arquivo principal:

sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bkp

7 - Configurar o Samba

Edite o arquivo de configuração:

sudo nano /etc/samba/smb.conf

Insira o seguinte conteúdo:

[global]
   workgroup = ESCRITORIO
   netbios name = debianserver
   server string = Servidor Samba Debian 13
   security = user
   map to guest = bad user

[Estoque]
   comment = Arquivos de usuários
   path = /arquivos/estoque
   public = no
   writable = yes
   guest ok = no
   read list = oliveira
   create mask = 0775
   directory mask = 0775

[Gestao]
   comment = Arquivos de usuários
   path = /arquivos/gestao
   browseable = no
   public = no
   writable = no
   guest ok = no
   write list = oliveira,giovanna
   create mask = 0775
   directory mask = 0775

[Publico]
   comment = Pasta Publico
   path = /arquivos/publico
   browseable = yes
   writable = yes
   guest ok = no
   public = yes
   create mode = 0777
   directory mode = 0777

8 - Ativar e reiniciar os serviços do Samba

No Debian 13, os serviços são os mesmos:

sudo systemctl enable smbd nmbd
sudo systemctl restart smbd nmbd

9 - Testar configuração

Verifique se não há erros:

testparm

10 - Acessar os compartilhamentos

Agora, de outra máquina na mesma rede, você pode acessar:

\\IP_DO_SERVIDOR\Estoque
\\IP_DO_SERVIDOR\Gestao
\\IP_DO_SERVIDOR\Publico

11 - Estrutura de Pastas e Permissões – Samba4 no Debian 13

/arquivos ├── estoque/ (rw para oliveira, somente leitura para outros) │ └── Permissões: │ - oliveira: leitura e escrita │ - demais: leitura │ ├── gestao/ (rw para oliveira e giovanna, bloqueado para outros) │ └── Permissões: │ - oliveira: leitura e escrita │ - giovanna: leitura e escrita │ - demais: sem acesso │ └── publico/ (rw para todos usuários Samba) └── Permissões: - Todos os usuários cadastrados: leitura e escrita

12 - Fluxo de Configuração (resumido)

Usuários criados no sistema │ ├──> oliveira └──> giovanna

↓ Usuários adicionados ao Samba smbpasswd -a oliveira smbpasswd -a giovanna

↓ Pastas configuradas em /arquivos estoque | gestao | publico

↓ Configuração no smb.conf [Estoque] [Gestao] [Publico]

↓ Serviços smbd + nmbd ativos
